# ChatGPT 代充平台

一个完整、可运行的 ChatGPT 代充服务网站，包含**用户端**与**管理端**，内置收款支付流程。

## 技术栈
- 后端：Node.js + Express（纯 JS，无需数据库服务）
- 存储：本地 JSON 文件（`data/` 目录，零配置、零编译）
- 前端：原生 HTML / CSS / JavaScript
- 认证：HMAC 签名 Cookie Token

## 快速开始

```bash
# 1. 进入项目目录
cd D:\代充网站

# 2. 安装依赖
npm install

# 3. 启动服务
npm start
```

启动后访问：
- 用户端：http://localhost:3000/
- 管理端：http://localhost:3000/admin/

### 默认账号
| 角色 | 用户名 | 密码 |
|------|--------|------|
| 管理员 | `admin` | `admin123` |
| 普通用户 | `demo` | `demo123` |

> ⚠️ 正式使用前请务必在「用户管理 / 数据库」中修改默认密码，并设置环境变量 `AUTH_SECRET`。

## 功能一览

### 用户端
- 注册 / 登录
- 浏览商品（Plus 会员、API 额度、Team 团队版、礼品卡，可按分类筛选）
- 下单（填写 ChatGPT 账号 / 联系方式 / 备注）
- 支付（扫收款码 + 上传凭证 / 或测试用模拟支付）
- 我的订单（查看状态、客服交付内容、取消待支付订单）

### 管理端
- 数据看板（订单、营收、用户统计）
- 商品管理（增删改查、上下架、上传图片）
- 订单管理（筛选、查看详情、修改状态、填写交付内容/备注、查看支付凭证）
- 用户管理（角色设置）
- 站点设置（站名、公告、联系方式、收款码、支付选项）

## 支付说明

本平台默认采用**收款码 + 人工核实**模式，这是个人/小型代充站最常用、**无需任何商户接口**即可真实收款的方式：

1. 管理员在「站点设置」上传支付宝 / 微信个人收款二维码；
2. 用户下单后扫码付款，并上传支付截图；
3. 管理员在「订单管理」核实后，将订单状态改为「处理中 / 已完成」，并可填写交付内容（如兑换码）。

### 接入在线自动支付（可选）
「站点设置 → 在线支付网关」预留了**易支付 / epay 兼容**接口参数（PID / KEY / 网关地址）。
如你拥有相应聚合支付商户账号，填写参数即可扩展为自动支付。对接逻辑可在 `src/routes/orders.js` 中实现回调验签。

> 接入支付宝 / 微信官方商户支付需企业资质与商户号，按其官方文档在后端补充下单与回调接口即可。

### 模拟支付（测试用）
设置中开启「模拟支付」后，支付页会出现「🧪 模拟支付成功」按钮，可一键把订单标记为已支付，用于在没有真实收款码时快速跑通完整流程。**正式上线请关闭。**

## 目录结构
```
代充网站/
├─ server.js              # 服务入口
├─ package.json
├─ src/
│  ├─ db.js               # JSON 文件存储
│  ├─ auth.js             # 认证（Token / 加密 / 中间件）
│  ├─ seed.js             # 初始化种子数据
│  └─ routes/             # API 路由
│     ├─ auth.js  products.js  orders.js  admin.js  settings.js
├─ public/                # 前端
│  ├─ index.html  login.html  register.html  orders.html
│  ├─ css/style.css   js/common.js
│  └─ admin/              # 管理后台页面
└─ data/                  # 运行时生成的数据与上传文件（勿提交）
```

## 安全与上线建议
- 设置 `AUTH_SECRET` 环境变量（用于 Token 签名）。
- 修改默认管理员密码。
- 关闭模拟支付。
- ChatGPT 账号密码等敏感字段建议在 `src/orders.js` 落库前加密。
- 生产部署建议置于 Nginx 反向代理后并启用 HTTPS。
- 自定义端口：`set PORT=8080 && npm start`（Windows）。
